Today, I decided to polish and create the review API. I added the additional fields needed for the flashcard algorithm and created the two simple get and create endpoints. There isn't a need for users to update or delete reviews so I omitted these. When it comes to the leader boards, it will rely upon this and I realise that the API can be abused. However, I will add rate limits and since leader boards are not global, it shouldn't affect users that much as they can unfriend the cheater.

I wrote the code and it all worked first time due to Rust's type system keeping me safe. I only have to be careful about my database definitions and conversations between the database response and the "cleaned" version.

Next, I have to implement the endpoints for specialised functions like the leader boards, adding friends, etc.
